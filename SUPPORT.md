# Support

 basement is open-source software (AGPL-3.0). Here's where to get help:

## Bugs

File a bug report using the [bug-report template](.github/ISSUE_TEMPLATE/bug_report.yml).
Include:

- What you expected vs. what happened
- Steps to reproduce
- Environment details (OS, Docker version, backend type)
- Logs or screenshots if relevant

See [`CONTRIBUTING.md`](./CONTRIBUTING.md) for the dev loop and how to sign off commits.

## Feature Requests

Start a [feature-request discussion](.github/ISSUE_TEMPLATE/feature_request.yml) or
open an issue with the feature-request template. We use GitHub Discussions for
early design feedback before implementation.

## Questions

If you have a usage question:

- Check [`docs/`](./docs/) — deployment, configuration, integrations, and more.
- Start a [question discussion](.github/ISSUE_TEMPLATE/question.yml) or browse
  existing discussions on the repo.
- Search existing issues — your question may already be answered.

## Security

**Do not file security issues in public.** See [`SECURITY.md`](./SECURITY.md).

Send reports to `matthew@pq.io`. We'll acknowledge receipt within 48 hours and
provide an update timeline for triage, fix, and disclosure.

## Commercial / Dual-Licensing

For proprietary embedding, hosted SaaS without source disclosure, or other use
cases the AGPL's copyleft doesn't suit: contact `matthew@pq.io`. The maintainer
reserves the right to grant commercial licenses on a case-by-case basis.
