# v1.12.0b: ADR-0007 Compliance Verification (Federation Uses UserRegion Secrets)

**Status**: Accepted  
**Date**: 2026-05-23  
**Build**: v1.12.0b  

## Summary

This cycle verifies that federation engine correctly uses **UserRegion JWT secrets** (not admin_tokens/CSK), confirming ADR-0007 Section "Scope: what CSK protects vs. what stays where it is" applies correctly in practice.

## Key Finding

Federation does NOT need CSK lock-state gates because:
1. Per ADR-0007, UserRegion S3 secrets remain **unchanged** — still JWT-derived (line 97 of `docs/adr/0007-per-cluster-envelope-encryption.md`)
2. Different trust model: the user logs in with their basement password (not a separate cluster admin password)
3. No migration to CSK planned for UserRegion secrets per ADR-0007 Section "Out of scope" (lines 236-239)

## Background Jobs Comparison

| Job | Needs Lock-Gate? | Why |
|---|---|---|
| Federation engine | NO | Uses UserRegion JWT secrets, not CSK (`docs/adr/0007-per-cluster-envelope-encryption.md:97`) |
| Backup runner | YES | Destination may use CSK-protected connections (per ADR-0007 migration) |

Backup runner lock-gate implementation at `internal/api/backup_runner.go:96-109` will be documented in v1.12.0c.
