# S3 API gateway

> **Status:** stub. The S3 API protocol is registered in basement's
> gateway registry so the `/admin/gateways` UI surfaces it as a
> placeholder, but no implementation ships today. basement currently
> routes S3 traffic via the user's region keychain (the user supplies
> an S3 endpoint + key per region; basement is the admin pane on
> top). A native S3 gateway would invert that: basement itself would
> terminate an S3 endpoint, verify SigV4 against service-account
> credentials, and proxy the request to the underlying driver &mdash;
> turning every existing S3 SDK (`boto3`, `aws-sdk-go-v2`,
> `@aws-sdk/client-s3`, `aws-sdk-s3`, &hellip;) into a first-class
> basement client. The gating pieces are a SigV4 request verifier and
> an S3-verb-to-Backend-verb proxy; contributions are welcome and the
> Gateway interface they'd plug into is documented in
> [adding-a-gateway.md](adding-a-gateway.md). Until then, point S3
> SDKs at a basement user-region S3 endpoint directly &mdash; the
> service-account credentials work and basement is transparent at
> the wire level.
