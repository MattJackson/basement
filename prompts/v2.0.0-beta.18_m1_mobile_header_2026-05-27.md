# v2.0.0-beta.18 — M1: PersonaPill overlaps Logo on mobile

## What you're producing

A single commit fixing the mobile (≤640px viewport) header overflow
where `<PersonaPill />` visually overlaps the `<Logo />` in
`frontend/src/shared/layout/AppShell.tsx`. ONE commit, ONE tag.

## Background

beta.16 mobile smoke screenshots show the persona pill "UI ADMIN
(ELEVATED) 14:59 left × M" rendered on top of the Basement logo's
wordmark at 375px wide. Visible at:

- `/tmp/basement-smoke-2026-05-27T12-09-21-221Z/mobile/E-files-home.png`
- `/tmp/basement-smoke-2026-05-27T12-09-21-221Z/mobile/E-admin-clusters.png`

The header layout (`AppShell.tsx:125-201`):

```
<header h-16>
  <div flex items-center justify-between gap-2>
    <div flex flex-1 min-w-0 gap-3 sm:gap-6>   <!-- left: Logo + nav -->
      <Logo />
      <nav> ... links ... </nav>
    </div>
    <div flex flex-shrink-0 gap-1 sm:gap-2>     <!-- right: pill + menu -->
      <PersonaPill />
      <UserMenu />
    </div>
  </div>
</header>
```

The right cluster has `flex-shrink-0` so it keeps its full content
width; on a 375px viewport with PersonaPill text "UI ADMIN
(ELEVATED) 14:59 left × M" + UserMenu avatar, the right cluster is
~280px wide, leaving only ~80px for Logo + nav on the left. The
left clusters' `min-w-0` allows it to compress to that, but the
Logo wordmark "Basement" still has some intrinsic width and ends
up behind the right cluster's flex item due to negative-space
collision.

## Fix approach (pick the LIGHTEST that works)

**Option A (preferred):** Make the PersonaPill compact on mobile.
Inside the PersonaPill component (around line 290 where the chip
JSX is built), replace the verbose "UI ADMIN (ELEVATED) | 14:55 left | ×"
content with:
- Mobile (`sm:hidden` wrapper): just a tight badge with the role
  initial (e.g. "A" for admin, "U" for user) and a subtle dot
  indicator if elevated. Tap target stays — clicking opens the
  UserMenu (or just shows full info as a tooltip).
- Desktop (`hidden sm:inline-flex`): keep the existing verbose form.

The simplest implementation: wrap each text element in
`className="hidden sm:inline"`, and add a small `<span className="sm:hidden">` showing just the role initial.

**Option B (fallback):** Drop the countdown ("14:59 left") on mobile.
This shrinks the chip width by ~80px which may be enough. Wrap the
countdown span in `className="hidden md:inline"`.

**Option C (last resort):** Hide the PersonaPill entirely on mobile.
Role state lives in the UserMenu dropdown anyway. Add
`className="hidden sm:inline-flex"` to the PersonaPill at its call
site in AppShell.tsx:199.

Use Option A. Document any tradeoffs you noticed in the commit
message.
