many secondary labels.

## Exact changes

### 1. Color-contrast fix (theme tokens)

In `frontend/src/index.css`:

**Line 35** (light theme):
```css
  --color-muted-foreground: hsl(240 3.8% 46.1%);
```
Change to:
```css
  --color-muted-foreground: hsl(240 5% 35%);
```

That bumps the muted-foreground darker so its contrast on the white
`--color-background` is comfortably above 4.5:1 (around 7:1).

**Line 68** (dark theme):
```css
  --color-muted-foreground: hsl(240 5% 64.9%);
```
Change to:
```css
  --color-muted-foreground: hsl(240 5% 72%);
```

Lighten dark-theme muted-fg so it's brighter against dark backgrounds.

### 2. Heading-order fix on worst-offending pages

The pages flagged with heading-order in the smoke (almost every /files
list page + admin pages). For each one, the pattern is usually:

```tsx
<h1>Page title</h1>
<p>Subtitle</p>
...
<h3>Section title</h3>  // <- skipped h2
```

Fix by either:
- (a) changing `<h3>` to `<h2>` (preferred when the section is at the
  same level as the page subtitle), OR
- (b) wrapping the page's main content in `<section><h2>...</h2></section>`
  so the h3 has an h2 parent.

The smoke flagged these pages with heading-order violations (from
findings):
- /files (My Regions)
- /files/keys
- /files/shares
- /files/syncs
- /files/backups
- /files/federated-buckets
- /files/webhooks
- /admin (redirects to /admin/clusters)
- /admin/system
- /admin/users
- /admin/clusters
- /admin/users/new

Fix ALL of them in the route files. Each fix is 1-3 lines. Use grep:

```bash
grep -rn "<h3" frontend/src/routes/files/ frontend/src/routes/admin/ | grep -v __tests__
```

For each h3 you find, check whether an h2 precedes it in the same page.
If not, either promote the h3 to h2 or insert a wrapping section with h2.

If a page has NO h1 at all (page-has-heading-one violations on
/share/$token and /files/syncs), that's a different rule — add a
proper `<h1>` to those pages.
