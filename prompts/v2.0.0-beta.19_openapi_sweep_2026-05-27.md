# v2.0.0-beta.19 — OpenAPI endpoint sweep tool

## What you're producing

A new script `scripts/openapi-sweep.ts` that walks every documented
path × method in `openapi/basement.yaml`, calls the live API with
appropriate auth, and reports any drift between spec and implementation
(status code mismatches, missing endpoints, shape divergence).

Then RUN it and write findings.

This is infrastructure, not a fix cycle. Goal: catch spec/impl drift
so future cycles can re-run quickly.

## Scope

- 66 paths in the spec, ~89 endpoint × method combos.
- For each: substitute path params with sensible defaults (real
  bucket/cluster IDs from a discovery pass), call with the correct
  auth state, verify response status matches one of the declared
  responses, check response body roughly matches the declared schema
  (best-effort — just key presence, not full validation).

## Implementation outline

`scripts/openapi-sweep.ts`:

```ts
#!/usr/bin/env node
// openapi-sweep.ts — walks every (path, method) in openapi/basement.yaml,
// calls the live API, and reports drift.
//
// Auth: logs in as matthew, elevates, sets activeRole=ui-admin.
// Path params: substitutes from a small discovery pass (real cluster ID,
//   first bucket, first SA, etc).
// Output: /tmp/basement/openapi-sweep-{ts}.json + a human-readable summary.

import { readFileSync, writeFileSync, mkdirSync } from "node:fs";
import { resolve, dirname, join } from "node:path";
import { fileURLToPath } from "node:url";

const __dirname = dirname(fileURLToPath(import.meta.url));
const SPEC = resolve(__dirname, "..", "openapi", "basement.yaml");

const BASE_URL = process.env.BASE_URL ?? "https://basement.pq.io";
const USERNAME = process.env.BUI_USERNAME ?? "matthew";
const PASSWORD = process.env.BUI_PASSWORD ?? "password";

// 1. Parse YAML (use the js-yaml package — already a Playwright dep transitive,
//    or do regex-based parsing of paths since the spec is well-formed enough).
//    Simpler: shell out to `python3 -c "import yaml, json; print(json.dumps(yaml.safe_load(open('${SPEC}')).get('paths', {})))"` and parse that.
// 2. Auth bootstrap: POST /auth/login, POST /auth/elevate, PUT /auth/active-role.
//    Store the session cookie value.
// 3. Discovery pass: GET /admin/clusters, /admin/buckets, /admin/service-accounts —
//    grab one real ID from each for path-param substitution.
// 4. For each (path, method):
//    a. Substitute {cid}, {id}, {bucket}, {key+}, etc with discovered defaults
//       (or "test-not-found" for paths where we expect a 404).
//    b. fetch(path, { method }) with cookies.
//    c. Check status code is in the spec's `responses` keys (200/201/204/400/401/403/404/etc).
//    d. If 200, fetch the response body and verify it's parseable JSON (or expected
//       MIME if not JSON).
//    e. Skip write methods (POST/PUT/PATCH/DELETE) by default — only run reads.
//       Optional flag --writes to enable.
// 5. Output: JSON dump with per-endpoint result + summary counts.
```

Key implementation notes:
- Use Node's built-in `fetch` (Node 24+ has it native).
- DO NOT call DELETE/POST/PATCH/PUT on any prod endpoint by default —
  the read pass is enough to catch spec drift. The `--writes` flag is
  future work.
- For endpoints requiring path params that we can't discover (e.g.
  `/admin/clusters/{cid}/buckets/{id}` if no buckets exist), substitute
  a sentinel like `does-not-exist` and assert the response is 404 (or
  whatever the spec declares for 404 path).
- Use a 5-second timeout per request.
- Be tolerant of 401/403 on endpoints requiring different roles than
  ui-admin — just report them as "auth-gated, expected".

## Anti-fabrication gates (NON-NEGOTIABLE)

- Every numeric / symbolic claim has a pasted-output citation
- Every existing-code reference cites `file:line`
- When the source doesn't say what you need, mark **OPEN** — DO NOT invent
- `git status --short` before commit
- `git -C <absolute-path>` always
- Stage by file name. NEVER `git add .` or `-A`
- NO `--no-verify` / `Co-authored-by` / `--force`
- DO NOT exit before pushing — escalate if blocked
- The sweep is READ-ONLY by default. NO write methods (POST/PUT/PATCH/DELETE)
  against the live operator deploy without `--writes` flag (which we do
  not pass in this cycle)

  drift issues, log path

## What failure looks like

- Sweep script errors → fix in scope or **ESCALATE**
- Live API returns 5xx for many endpoints → likely deploy issue,
  **ESCALATE** with the count + sample
- Spec parse fails → check yaml structure; **ESCALATE** if format
  changed unexpectedly
