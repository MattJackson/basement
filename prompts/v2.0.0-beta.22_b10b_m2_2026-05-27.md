## Background

B10b: beta.20 added error detail to the swallowed 500. The detail surfaced
`reading sync directory: open /var/lib/basement/syncs: no such file or
directory`. Backend should treat a missing directory as an empty list
(no syncs configured yet). The directory only exists after a sync has
been saved. This is the right fix because it matches how absent storage
is treated elsewhere (e.g. empty buckets returns `[]`, not 500).

M2: post-beta.18 mobile screenshot shows the persona pill correctly
compact, but the LEFT cluster (Logo + nav links) is still cramped —
the "Basement" wordmark abuts the first nav link with only a 12px gap.

## Exact changes

### B10b fix (Go)

Option 1 (preferred — fix at store): in the SyncStore's `List` method,
wrap the file/dir read so `os.IsNotExist(err)` returns `nil, nil`
(empty list, no error). Then the handler doesn't change.

If the store's API doesn't currently let it return nil/empty cleanly
(e.g. it tries to read the directory first then panics), use Option 2.

Option 2 (fallback — fix at handler): in `internal/api/user_syncs.go`
around line 191:

```go
jobs, err := s.syncStore.List(claims.UserID)
if err != nil {
    if errors.Is(err, fs.ErrNotExist) || strings.Contains(err.Error(), "no such file or directory") {
        jobs = nil // treat empty store as empty list
    } else {
        slog.Error("listSyncsHandler: syncStore.List failed", "user", claims.UserID, "err", err)
        writeErrorSimple(w, http.StatusInternalServerError, "SYNC_STORE_ERROR",
            fmt.Sprintf("Failed to list sync jobs: %v", err))
        return
    }
}
```

(`io/fs` is already in stdlib; you may need to add the imports.)

Add a test in `internal/api/user_syncs_test.go` (extend the existing
file from beta.20) that:

- Asserts 200 + body is `[]` (or `null`).
