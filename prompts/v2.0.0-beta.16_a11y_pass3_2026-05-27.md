## Strategy notes

This is a LOT of mechanical work. Don't try to fix every single
violation perfectly — aim for the highest-frequency patterns first:

1. **heading-order** — read `/tmp/basement/a11y-detail-*.json`,
   filter `rule == "heading-order"`, group by `route`. For each
   affected page, find the first h3-after-h1-without-h2 and either:
   - Change h3 → h2 (preferred), OR
   - Add a wrapping `<section><h2>section title</h2>...</section>`
   - Cite the file:line where you make the change

2. **page-has-heading-one** — 4 routes lack any h1. Add a top-level
   h1 (could be visually-hidden via `className="sr-only"` if the
   page intentionally has no visible title).

3. **landmark-one-main + region** (both /admin/login) — wrap the
   sign-in card in `<main className="...">`. The existing wrapper
   `<div className="flex min-h-screen items-center justify-center">`
   in `frontend/src/shared/auth/LoginForm.tsx` is the place to make
   it `<main>`.

4. **empty-table-header** — find any `<th>` with no text content
   (likely action columns). Add `<th><span className="sr-only">Actions</span></th>`.

5. **select-name** — find each `<select>` lacking a label. The
   pattern in this codebase uses `<Label htmlFor="...">`+`<select id="...">`.
   Where missing, add the htmlFor/id pairing OR `aria-label="..."`.

6. **nested-interactive** — find any `<button>` containing another
   `<button>` or `<a>` containing a `<button>`. Restructure by
   moving the inner interactive element to a sibling or extracting
   to a separate row.

7. **color-contrast (remaining 12)** — re-run the detail dump if
   needed. Likely a small set of specific elements (icon-only buttons,
   ghost text, etc). Fix per-element via Tailwind class tweak or
   inline style.

**Scope cap:** if any single rule looks like it'd take more than
~10 file edits, skip it and document in findings.md. Better to land
clean fixes for 5 rules than half-finished work for 8.
