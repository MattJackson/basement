## Root cause (multiple)

### A. Wrong chi group for most /admin/clusters/{cid}/* routes

`internal/api/server.go` around line 422:

```go
adminG.Group(func(clusterG chi.Router) {
    clusterG.Use(auth.ActiveRoleClusterMiddleware("{cid}"))
    clusterG.Get("/admin/clusters/{cid}/nodes", s.listNodesHandler)         // gated (broken — see B)
    adminG.Get("/admin/clusters/{cid}/layout", s.getLayoutHandler)          // <- adminG, NOT gated
    adminG.Post("/admin/clusters/{cid}/layout/stage", s.stageLayoutHandler) // <- adminG, NOT gated
    // ... ALL the rest call adminG.* inside the clusterG closure ...
    adminG.Get("/admin/clusters/{cid}/keys", s.listKeysByClusterHandler)    // <- NOT gated
    adminG.Get("/admin/clusters/{cid}/buckets", s.listBucketsByClusterHandler)  // <- NOT gated
}) // end cluster admin group with active role middleware
```

Only `listNodesHandler` actually attaches to `clusterG`. Everything else bypasses the middleware and inherits only `adminG.Use(auth.RequireRole("admin"))` which checks the JWT role claim (which for any UI admin is "admin" regardless of active mode).

### B. ActiveRoleClusterMiddleware uses literal "{cid}" not the URL param

`internal/auth/active_role.go`:

```go
func ActiveRoleClusterMiddleware(cid string) func(http.Handler) http.Handler {
    return ActiveRoleMiddleware([]string{"cluster-admin"}, cid)
}
```

Called as `ActiveRoleClusterMiddleware("{cid}")`. The "{cid}" string is passed as `requiredCluster` and the middleware compares `ar.Cluster != "{cid}"` literally. NEVER matches. Result: even legitimate cluster-admins get 403 on `/admin/clusters/{cid}/nodes`.

### C. No defense in depth on adminG

The whole adminG group has `RequireRole("admin")` (JWT-level) but NO active-role gate. v1.13.28 added the unused `ActiveRoleAnyAdminMiddleware()` helper ready for this cycle.

## Fix (in this order)

### 1. Add a dynamic per-cluster active-role middleware (`internal/auth/active_role.go`)

```go
// ActiveRoleClusterMiddlewareFromPath reads the cluster ID from the chi
// URL param "cid" at request time and checks activeRole.Kind == "cluster-admin"
// AND activeRole.Cluster == cid. UIAdmin active role also passes (super-admin).
func ActiveRoleClusterMiddlewareFromPath() func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            claims, ok := FromContext(r.Context())
            if !ok || claims == nil || claims.ActiveRole == nil {
                http.Error(w, `{"error":{"code":"UNAUTHORIZED","message":"No active role"}}`, http.StatusUnauthorized)
                return
            }
            ar := claims.ActiveRole
            // UI Admin is super-admin — passes any cluster route
            if ar.Kind == "ui-admin" {
                next.ServeHTTP(w, r)
                return
            }
            if ar.Kind != "cluster-admin" {
                http.Error(w, `{"error":{"code":"FORBIDDEN","message":"Active role not permitted for this route"}}`, http.StatusForbidden)
                return
            }
            cid := chi.URLParam(r, "cid")
            if cid == "" || ar.Cluster != cid {
                http.Error(w, `{"error":{"code":"FORBIDDEN","message":"Cluster admin scope mismatch"}}`, http.StatusForbidden)
                return
            }
            next.ServeHTTP(w, r)
        })
    }
}
```
