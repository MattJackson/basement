## Exact changes

### T1 — Pill label

In `PersonaPill.tsx`, find where `roleLabel` is computed
(around line 327-332):

```tsx
const roleLabel = activeRole.kind === "cluster-admin" && activeRole.cluster
  ? `Cluster Admin: ${activeRole.cluster}`   // <- this is the UUID
  : activeRole.kind === "ui-admin"
    ? "UI Admin"
    : "User";
```

Replace with a lookup that uses `availableRoles[i].label`:

```tsx
const matchedRole = availableRoles.find(
  (r) => r.kind === activeRole.kind &&
         (r.kind !== "cluster-admin" || r.cluster === activeRole.cluster),
);
const roleLabel = matchedRole?.label
  ?? (activeRole.kind === "ui-admin" ? "UI Admin"
    : activeRole.kind === "cluster-admin" ? "Cluster Admin"
    : "User");
```

The fallback chain handles the edge case where availableRoles hasn't
loaded yet (during initial mount): use the kind directly, drop the
cluster UUID rather than show it.

### T2 — Drop Buckets link for cluster-admin

In `AppShell.tsx`, the "Buckets" link is currently outside the
`{isUIAdminActive && (...)}` block (line 150-156). MOVE it inside that
block so it only renders for ui-admin. Or wrap it in a separate
conditional `{isUIAdminActive && (<Link to="/admin/buckets">...)}`.

Don't add ANY top-nav link for cluster-admin — the cluster detail
page has its own sub-nav. Per the operator: top nav can be empty/minimal
for cluster-admin.

### T3 — Redirect cluster-admin from /admin/clusters

In `ProtectedRoute.tsx`, after the existing `isClusterScopedRoute`
branch (or right at the start of the "other admin route" branch),
add:

```ts
// Cluster admin only has ONE cluster — bounce them from the list
// page to their cluster's detail page so the 403 from the
// cross-cluster aggregate endpoint never surfaces.
if (
  pathname === "/admin/clusters" &&
  activeRole?.kind === "cluster-admin" &&
  activeRole.cluster
) {
  navigate({ to: `/admin/clusters/${activeRole.cluster}` });
  return;
}
```

Match the exact pathname `/admin/clusters` (no trailing slash, no segment).

### T4 — Hide "Back to Clusters" link for cluster-admin

  <Link to="/admin/clusters">← Back to Clusters</Link>

(The Link import is likely already there. If `useUser` isn't imported,

Also check `frontend/src/routes/admin/clusters/new.tsx:104` for
