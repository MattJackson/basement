## Tasks

### 1. Set up i18n config

Create `frontend/src/shared/i18n/index.ts`:
- Initialize i18next with the browser language detector
- Configure 2 namespaces to start: `common` (buttons, errors, navigation) and `pages` (page-specific)
- Fallback language: `en`
- Initial supported languages: `en` + `es`. Code the supported-languages list as a typed array exported from `frontend/src/shared/i18n/index.ts` (this list will grow to ~20-30 LTR locales by v2.0 final — design the structure for that scale now, even though only 2 entries today)
- Detection order: localStorage → navigator language → fallback
- React Suspense integration for async loading

### 2. Wire provider into main.tsx

`frontend/src/main.tsx`:
```typescript
import "@/shared/i18n";
```
Before the ReactDOM render. The default export self-initializes — no provider wrapper needed (react-i18next uses context internally).
