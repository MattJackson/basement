# Cycle: v1.13.1 — Skin admin redesign + activation fix

Operator-locked redesign of the skin admin UX. The current per-row Activate/Delete UI is confusing + the activate flow doesn't reliably re-render the page with the new skin. Replace with three top-level controls + a simple installed-skins list.

## New UX (admin's /admin/system → Skins section)

```
Default Skin:      [basement-default ▾]
User Overridable:  [✓]   ← toggle
Available Skins:   ☑ basement-default     ☑ basement-95
                   ☑ basement-minimal     ☑ basement-terminal
                   ☐ basement-high-contrast
                   ☑ acme-corp (uploaded)
   (hidden when User Overridable = N)

Installed skins (preview + delete uploaded):
  [swatch] basement-default       built-in
  [swatch] basement-95            built-in
  [swatch] basement-minimal       built-in
  [swatch] basement-terminal      built-in
  [swatch] basement-high-contrast built-in
  [swatch] acme-corp              [Delete]
```

## OrgCapabilities migration

Replace:
- `SkinPolicy string` (was "default" | "locked" | "user-choice")

With:
- `UserOverridableSkin bool` (default false)
- `AllowedUserSkins []string` (default empty; when UserOverridable=true with empty list, falls back to "all installed")

Legacy migration (in OrgCapabilities loader):
- `SkinPolicy == "default"` or `"locked"` → `UserOverridableSkin = false; AllowedUserSkins = []`
- `SkinPolicy == "user-choice"` → `UserOverridableSkin = true; AllowedUserSkins = []` (all)
- Drop `SkinPolicy` field from the JSON shape; ignore on read if present

## Fallback semantics

- User Overridable = false → everyone gets ActiveSkin. User per-user skin override ignored on render.
- User Overridable = true + AllowedUserSkins non-empty:
  - User has picked a skin in the allowed set → user gets that skin
  - User picked a skin NOT in the allowed set (admin removed it) → fall back to ActiveSkin
  - User hasn't picked → fall back to ActiveSkin
- User Overridable = true + AllowedUserSkins empty → treat as "all installed skins available"
