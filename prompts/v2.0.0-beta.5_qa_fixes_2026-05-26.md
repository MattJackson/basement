## Read these first (absolute paths, with why)

- `/Users/mjackson/Developer/basement-ui/frontend/src/shared/auth/LoginForm.tsx`
  — see line 121. The `<CardDescription>{t("auth.username")} {t("auth.password")}</CardDescription>`
  is a v2.0.0-beta.4 i18n extraction bug. Renders as literal "Username Password"
  in the live UI. There's no real description copy in en/pages.json for this slot
  (`authLogin.signInTitle` is the title; no `signInSubtitle` exists). Resolution:
  delete the CardDescription line entirely. Do NOT add a placeholder. The card
  already has a clear title.

- `/Users/mjackson/Developer/basement-ui/frontend/src/routes/admin/system.tsx`
  — see line 169:
  ```tsx
  value={data.signupMode ?? "invite"}
  ```
  The live API returns `signupMode: ""` (empty string). `??` only catches
  null/undefined, so `""` passes through. The HTML select then renders with
  no value match. Change `??` to `||` so empty string also falls through to
  the default. Confirmed live behavior via `curl https://basement.pq.io/api/v1/admin/system`.

- `/Users/mjackson/Developer/basement-ui/scripts/comprehensive-smoke.ts`
  — see lines ~411-420 (elevate block) and ~430-440 (reap block).
  Two edits:

  (a) After each `elevateToAdmin(desktop!)` / `elevateToAdmin(mobile!)` call,
  add a PUT `/api/v1/auth/active-role {kind:"ui-admin"}`. Without this,
  v1.13.29 gating bounces /admin/* page navigations to /files and blocks
  POST /admin/* writes with 403. There are TWO elevate sites in the file:
  one in [0] bootstrap (around line 411) and one in [E] mobile (search for
  `elevate mobile context to admin`, ~line 654).

  (b) In the reap loop (line ~436), the filter is:
  ```ts
  (it.name ?? "").startsWith("smoke-")
  ```
  Change it to:
  ```ts
  /^(smoke|feat-smoke)-/.test(it.name ?? "")
  ```
  60 stranded `feat-smoke-*` service accounts are currently stuck in prod
  /admin/service-accounts because the reap doesn't match them.

- `/Users/mjackson/Developer/basement-ui/frontend/src/routes/admin/login.tsx`
  — currently:
  ```tsx
  if (typeof window !== "undefined") {
    window.location.replace("/login");
  }
  throw new Error("Redirect to /login");
  ```
  Replace the `throw new Error(...)` with a TanStack `throw redirect({ to: "/login" })`.
  Import `redirect` from `@tanstack/react-router`. The `window.location.replace`
  line should also go away — `redirect()` is the canonical pattern. The reason
  the existing code threw an Error is documented in the comment ("doesn't support
  custom status codes") — that comment is stale; `redirect()` is the right tool.
